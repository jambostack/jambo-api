III. Écarts restants — Ce qui manque encore

✅ 1. Moteur d'automatisation no-code (« Flows ») — FAIT v1.13

Implémenté : 48 handlers (6 triggers, 7 logic, 7 action, 2 HTTP, 5 AI, 3 Database, 3 File, 9 Transform, 6 Utility). FlowBuilder visuel avec ReactFlow, Zustand store, undo/redo, dry-run, validation DAG, i18n 4 langues.

48 handlers découverts dans src/Service/Flow/Handlers/ — livré avec 2 jours d'avance sur le planning.

✅ 2. Champs conditionnels & validation avancée — FAIT v1.10

Implémenté : Entité Field avec validationRules JSON. Service FieldConditionEvaluator. Composant ConditionalFieldWrapper. Validateurs client TypeScript (validators.ts). Unicité serveur. Feedback validation temps réel.

Manque mineur : l'éditeur visuel de conditions dans FieldFormModal (les conditions fonctionnent via options JSON, mais pas d'UI dédiée).

✅ 3. Dossiers médias + uploads résumables (TUS) — FAIT v1.11

Implémenté : Entité MediaFolder avec arborescence parent/children. API CRUD complète. Composant MediaFolderTree avec création/renommage/suppression inline. Protocole TUS 1.0 avec TusServer, chunks, reprise, sécurité (path-traversal, file-type).

✅ 4. Realtime / SSE / GraphQL Subscriptions — FAIT v1.12b

Implémenté : Hub Mercure SSE double canal (HubInterface + JSONL fallback). Endpoint GET /realtime/token (JWT Mercure). useRealtime EventSource prioritaire + fallback polling. GraphQL Subscription type avec types ProjectEvent/ContentEvent. Cookie mercureAuthorization au login admin. MercureCookieSubscriber.

Commits : 10 commits (e2e641c..2d0c4d0). Config Docker Mercure sur port 8080 + cors_origins.

✅ 5. Live Preview + Visual Editing — FAIT v1.14a/v1.14b

Implémenté v1.14a : iframe + postMessage bridge. JWT preview token (PreviewTokenService, HS256). Endpoint /preview/token/{entryUuid} et /preview/content/{col}/{eid}. Composant LivePreviewPanel (device switching, loader, error badge). SDK npm @jambostack/live-preview (core + Next.js). Migration preview_url, preview_mode, preview_enabled sur Project.

Implémenté v1.14b : Survol/click-to-edit via data-jambo-field. initVisualEditing() (MutationObserver, popover inline). Highlight + scrollIntoView dans ContentForm. fieldProps() dans le hook Next.js. 5 messages postMessage visuels.

Commits : 9 commits v1.14a + 5 commits v1.14b. Zéro backend pour v1.14b.

🟠 6. Système d'extensions / plugins

Concurrents : Les trois ont une architecture de plugins.

Jambo : Aucun mécanisme d'extension. Le NodeRegistry des flows est le seul pattern d'extensibilité (auto-discovery via compiler pass), mais pas un système générique.

Impact : ⭐⭐⭐ — Structurant pour l'écosystème. Effort estimé : 10-15 jours.

🟡 7. Dashboards, Partages, SAML/OIDC, SEO, Form Builder

Dashboards : Page dashboard basique (liste des projets). Pas d'insights, graphiques, KPI.

Partages publics : Aucun code.

SAML/OIDC : OAuth social (4 providers) présent, mais pas SAML/OIDC fédéré.

SEO/Redirects/Form Builder : Aucun code.

Impact : ⭐⭐ — Effort estimé : 5-7 jours.
