01 Vision

Jambo Workbench est un atelier de développement web piloté par IA, dans le navigateur. L'utilisateur décrit en langage naturel le site voulu ; l'agent génère le code, l'exécute en direct (WebContainer), et l'utilisateur prévisualise, édite, itère, puis déploie.

La différence clé avec bolt.diy : les sites sont alimentés par les collections d'un projet Jambo. On connecte son projet (URL d'API + token), l'agent reçoit le schéma des collections et génère un site data-driven qui consomme l'API Jambo (REST/GraphQL).

Principe directeur : rapide et fonctionnel. Tout ce qui freine l'itération (builds lents, crashs, friction) est un bug.

02 Ce qu'on emprunte à bolt.diy

À reprendre

  • Agent conversationnel qui écrit/modifie des fichiers en streaming, avec un parser d'artefacts (actions « écrire fichier » / « lancer commande »).
  • WebContainer (@webcontainer/api) : Node dans le navigateur → install + dev server + preview, sans backend.
  • Layout Workbench : chat à gauche, espace de travail à droite (onglets Code / Preview / Terminal), panneaux redimensionnables.
  • Multi-LLM via le Vercel AI SDK : OpenAI, Anthropic, Google, Mistral, DeepSeek, Ollama, OpenRouter…
  • CodeMirror 6, terminal xterm.js, diff, export ZIP, import/export GitHub.
  • Snapshots / historique de versions, rollback.

Ce qui diffère (valeur Jambo)

  • Connexion Jambo : projet, schéma des collections, injection dans le system prompt.
  • L'app générée consomme l'API Jambo via un client généré lib/jambo.ts (listes, fiches, pagination, filtres, i18n).
  • Templates data : blog, catalogue, vitrine, portfolio, landing.
  • v2 : mode SSR où le rendu interroge Jambo côté serveur.

04 Architecture

jambo-workbench/ ├─ vite.config.ts # react, node-polyfills ; headers COOP/COEP (WebContainer) ├─ tailwind.config.ts ├─ src/ │ ├─ main.tsx │ ├─ app/ # Home, Workbench, Settings │ ├─ components/ │ │ ├─ chat/ # ChatPanel, Message, PromptInput │ │ ├─ workbench/ # FileTree, CodeEditor, Preview, Terminal, ResizableLayout │ │ ├─ jambo/ # ProjectPicker, CollectionsBrowser, ConnectDialog │ │ └─ ui/ # shadcn/ui │ ├─ lib/ │ │ ├─ ai/ # providers, streamText, system prompts, parser d'artefacts │ │ ├─ webcontainer/ # boot, mount, run, writeFile, terminal │ │ ├─ jambo/ # client API (token, getCollections, getEntries) │ │ ├─ git/ # isomorphic-git + octokit │ │ └─ stores/ # nanostores: chat, files, workbench, jambo, settings │ ├─ runtime/ # templates de départ + générateur lib/jambo.ts │ └─ styles/ └─ docs/

Flux de données

  • Connexion Jambo → token + apiUrl (settings) → schéma des collections.
  • Prompt → system prompt enrichi du schéma + framework → streamText (provider choisi).
  • Streaming → parser d'artefacts → writeFile (store + WebContainer) + commandes shell.
  • WebContainer → install + dev → URL de preview (iframe).
  • Itération → édition CodeMirror re-synchronise ; snapshots auto.
  • Sortie → ZIP, push GitHub (octokit), ou déploiement.

05 Fonctionnalités

MVP

  • Connexion à un projet Jambo (apiUrl + token) ; lecture du schéma.
  • Chat IA multi-provider en streaming + sélecteur provider/modèle.
  • Parser d'artefacts (fichiers + commandes) appliqués en direct.
  • WebContainer : install + dev server + preview.
  • Workbench : arbre de fichiers, CodeMirror, terminal xterm, onglets redimensionnables.
  • Templates de départ (Vite+React, Astro, Next) + lib/jambo.ts généré.
  • Snapshots auto + rollback ; export ZIP.

v1

  • Import/Export GitHub (octokit), clone de repo.
  • Déploiement 1-clic (Netlify / Vercel / Cloudflare Pages).
  • Détection d'erreurs de build + bouton « Corriger » (auto-fix).
  • Multimodal (images dans le prompt) selon provider.
  • Settings : clés providers, thème, préférences.

v2

  • Mode SSR/edge (le site interroge Jambo côté serveur).
  • Composants Jambo prêts (listes, fiches, filtres, pagination, i18n).
  • App desktop Electron.

06 UX & Layout

  • Écran façon bolt : chat à gauche (~38%), workbench à droite (onglets Aperçu / Code / Terminal), séparateurs react-resizable-panels.
  • Loader de génération inline en bas du fil (pas dans l'en-tête).
  • Barre d'actions collante (erreurs → « Corriger ») juste au-dessus du composer.
  • État vide : suggestions de prompts + choix template + projet Jambo.
  • Thème sombre par défaut, animations sobres (framer-motion).

07 Intégration Jambo (cœur de la valeur)

Client lib/jambo

  • Auth : Authorization: Bearer <API_TOKEN> (token de projet).
  • Endpoints : liste des projets, schéma (collections + champs + types), entrées (pagination, locale, filtres), entrée par slug.
  • Client typé : getCollections(), getEntries(slug, opts), getEntry(slug, entrySlug).

Injection dans l'IA

Le system prompt inclut le schéma des collections, le framework cible, le format d'artefacts, et la consigne d'utiliser lib/jambo.ts (jamais d'URL en dur) avec pagination/i18n.

Générateur lib/jambo.ts

Inséré dans chaque template : client prêt-à-l'emploi lisant import.meta.env.JAMBO_API_URL + token, exposant les fonctions ci-dessus côté app générée.

08 Sécurité

  • Tokens : stockés côté client (chiffré) ou via mini-proxy ; jamais committés.
  • WebContainer : sandbox navigateur. Iframe de preview sans allow-same-origin si du JS arbitraire peut s'y exécuter.
  • Clés providers : saisies utilisateur, locales ou proxifiées ; jamais dans le bundle.
  • CORS Jambo : prévoir un mode proxy si l'API n'autorise pas l'origine du Workbench.

09 Design system

  • Typo : display caractériel (Space Grotesk, General Sans, Clash Display) + corps lisible (Geist, IBM Plex, Satoshi). Mono : JetBrains / Geist Mono.
  • Couleurs : base sombre (anthracite), accent franc (violet électrique ou vert acide), en variables CSS.
  • Composants : shadcn/ui sur Tailwind (Button, Tabs, Dialog, Switch, Tooltip, ScrollArea, Resizable).
  • Motion : apparition échelonnée des messages, transitions fluides des panneaux, micro-interactions.

12 Critères d'acceptation (MVP)

  • Connecter un projet Jambo et voir ses collections.
  • Décrire un site → l'agent génère → preview live en < 30 s après install.
  • Éditer un fichier → la preview se met à jour.
  • Le site généré affiche de vraies données d'une collection Jambo.
  • Export ZIP fonctionnel.
